查询的过程：全表扫描，索引扫描

索引扫描：
索引表有两字段：被索引的字段，和对应的物理地址
过程： 找到索引表 -> 找到满足条件的字段的物理地址 -> 在存储文件中找到对应的记录

------------------分类
B*树索引：唯一索引：被索引的字段不能存在重复值
	非唯一索引：

位图索引：主要针对大量相同值的字段而创建
	索引表：被索引的字段（无重复值），起始rowid，结束rowid，位图向量（1代表有，0代表无）。

------------------语句
create [unique | bitmap] index 索引名
on 表名(字段名1 [ASC | DESC],. . .）

解释：
unique:创建唯一索引，不指明为非唯一索引
bitmap:创建位图索引

注意：
当索引的字段有多字段时，查询条件所有字段都在其中则引用索引；
只有其前导字段筛引用索引；
只有一个非前导字段时，前导索引字段索引值重复值多，则引用；

-----------------查看索引
索引信息存放在数据字典中：DBA_INDEXES,USER_INDEXES,USER_IND_COLUMNS

//查看表的索引名
select index_name 		
from USER_INDEXES 
where table_name='表名'

//查看所有是否被引用
analyze table 表名 compute statictics;	//查询计划分析
set autotrace on explain;		//打开计划分析
执行select,即可看到是否被引用
